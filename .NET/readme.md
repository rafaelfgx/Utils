# .NET and C#

## What is .NET?

.NET is a free, open-source development platform from Microsoft that provides a robust environment for running applications written in C#, F#, VB.NET, and other languages. In addition to the CLR (Common Language Runtime), it includes libraries and tools essential for software development.

## What is C#?

C# is a modern, object-oriented, and secure programming language developed by Microsoft. It is compiled to the CLR of .NET, providing a reliable and efficient runtime environment.

## What is CLR (Common Language Runtime)?

CLR is the execution core of .NET, responsible for managing code execution. It provides services such as memory management, assembly loading, exception handling, and security implementation.

## What is IL (Intermediate Language)?

IL is the intermediate code generated by the C# compiler. It is a virtual machine code that can be executed on any platform that implements the CLR.

## What is the purpose of the obj and bin directories?

* **obj:** Stores intermediate files generated by the compiler to assist in building the final program. It includes metadata, temporary compiled objects, and other artifacts. These are not directly executable.

* **bin:** Stores files in the final format that can be executed directly or used by other programs. Examples: executables (.exe, .dll), libraries (.lib).

## What is the difference between Stack and Heap Memory?

### Stack

* **Stores:** Value types (int, float, boolean, etc.) and local variables of functions.
* **Management:** Automatic via the call stack (stack frame).
* **Allocation:** Sequential, as functions are called.
* **Deallocation:** Automatic when leaving the function (variable scope ends).
* **Advantages:** Fast data access, as the memory location is directly calculated based on the stack position.
* **Disadvantages:** Limited space, not suitable for large or long-term allocations.
* **Example:** Temporary variables inside a function, like a loop counter.

### Heap

* **Stores:** Reference types (pointers to objects) and dynamically allocated objects.
* **Management:** By the garbage collector, which frees memory when objects are no longer used.
* **Allocation:** Dynamic, using operators like new or malloc.
* **Deallocation:** Manual, through the delete operator or when the garbage collector identifies that the object is no longer accessible.
* **Advantages:** Flexibility for allocating and deallocating large amounts of memory, ideal for complex data structures and long-lived objects.
* **Disadvantages:** Slower data access, as the pointer needs to be dereferenced to get the actual object.
* **Example:** Dynamic arrays, linked lists, objects created with new.

## What is Garbage Collector?

Garbage Collector is a crucial component of the CLR that automatically manages memory, allocating and freeing it when objects are no longer in use to avoid memory leaks.

Objects are organized into three main generations based on their lifespan and usage patterns:

### Gen0 (Generation 0):

* New objects are allocated for the first time.
* Short-lived memory area where objects are quickly collected.
* Many objects are allocated and deallocated quickly.
* Most objects do not survive a Gen0 collection.

### Gen1 (Generation 1):

* Objects that survive a Gen0 collection are promoted to Gen1.
* Contains objects with a somewhat longer lifespan.
* Gen1 collection occurs less frequently than Gen0, but still relatively quickly.

### Gen2 (Generation 2):

* Objects that survive a Gen1 collection are promoted to Gen2.
* Long-lived objects generation, containing objects that exist for an extended period during execution.
* Gen2 collection is slower and less frequent, as it involves deeper object inspection.

## What is Boxing and Unboxing?

* **Boxing:** Conversion of a value type (int, string, etc.) into a reference type (object). Useful when a value needs to be treated as a generic object. Example: Storing a value in a collection that accepts only objects.
* **Unboxing:** The opposite of boxing, converting a reference type back into a value type. It accesses the original value stored inside the object. Example: Accessing the value from an object stored in a collection.

## What is the difference between Class, Struct, and Record?

* **Class:** A reference type that supports inheritance, polymorphism, and is managed by pointers.
* **Struct:** A value type that is more memory-efficient and allocated on the Stack.
* **Record:** Similar to Struct, but optimized for immutability, with simplified constructors and value-based equality.

## What is the difference between Abstract Class and Interface?

* **Abstract Class:** Defines methods and properties that can be implemented or inherited by derived classes, and may contain both concrete and abstract methods.
* **Interface:** Defines methods and properties that must be implemented by any class that uses it, without providing any implementations.

## What is a Sealed Class?

A Sealed class is a class that cannot be inherited, preventing other classes from extending its functionality.

## What is a Static Class?

A Static class is a class that can only contain static members and cannot be instantiated.

## What is an access modifier?

Defines the visibility of classes, class members, and enumerations, such as public, private, protected, and internal.

## What is a Delegate?

A Delegate is a type-safe representation of a function pointer, allowing methods to be stored in variables, passed as parameters, or invoked asynchronously.

## What is a Multicast Delegate?

A multicast delegate allows multiple methods to be called sequentially when the delegate is invoked, using operations like += to add methods and -= to remove them.

## What is an Event?

Events are based on delegates and allow a class to notify other objects about the occurrence of actions or state changes.

## What is Reflection?

Reflection is the ability of a program to inspect and manipulate type metadata at runtime, allowing for introspection of objects and dynamic method invocation.

## What is Lambda?

Lambda is an anonymous function that can be used to create delegates or expression tree expressions concisely, without needing to define a separate method.

## What is the difference between ref and out operators?

* **ref:** Passes a reference of a variable to a method, allowing the method to modify the original variable's value.
* **out:** Used to return multiple values from a method.

## What is the difference between IEnumerable, ICollection, IList, List<T>, and IQueryable?

* **IEnumerable:** The base interface defining the functionality to enumerate items in a collection.
* **ICollection:** An interface that inherits from IEnumerable and adds functionalities such as counting items, adding items, removing items, and checking if an item is present.
* **IList:** An interface that inherits from ICollection and adds index-based access functionalities: inserting items at any position, removing items by index, accessing items by index.
* **List<T>:** A concrete class that implements IList<T>, providing a generic and efficient list implementation.
* **IQueryable:** Represents a query that can be executed on a data source, usually translated into SQL for database queries.

## What is async/await?

Async/await is a syntax in C# that simplifies writing asynchronous code, making it easier to develop operations that depend on runtime, such as I/O or API calls.

## What is the difference between Thread, Task, and ValueTask?

* **Thread:** A unit of execution at the operating system level.
* **Task:** Represents an asynchronous operation that may or may not return a specific value and supports cancellation and continuation.
* **ValueTask:** A lightweight structure representing an asynchronous operation returning a single value.

## What are the four basic principles of OOP?

* **Abstraction:** Focus on essential characteristics, hiding unnecessary details.
* **Encapsulation:** Protection of an object's internal data, allowing controlled access through methods.
* **Inheritance:** Reuse and extension of functionality through class hierarchies.
* **Polymorphism:** The ability of an object to take multiple forms and behaviors through interfaces and inheritance.

## What is the advantage and disadvantage of an Index?

* **Advantage:** Faster data retrieval.
* **Disadvantage:** Higher storage space and slower data insertion and updating.

## What is the difference between Clustered Index and Nonclustered Index?

### Clustered Index

* Stores table data and the index in the same physical structure.
* Organizes data in the order of the indexed column.
* Only one per table.
* Ideal for queries that retrieve multiple records based on an indexed column.
* Example: Index on the primary key of a table.

### Nonclustered Index

* Stores the index in a structure separate from the table data.
* Does not affect the physical order of data.
* Can be created on multiple columns.
* Useful for queries filtering or sorting by columns that are not the primary key.
* Example: Index on a column used to filter results.

## What is the concept of FIFO (First In, First Out) and LIFO (Last In, First Out)?

* **FIFO:** A data structure where the first element inserted is the first to be removed, like in a queue.
* **LIFO:** A data structure where the last element inserted is the first to be removed, like in a stack.

## What is the difference between Topic and Queue?

* **Topic:** A message channel where a message can be published and delivered to multiple subscribers.
* **Queue:** A message channel where messages are processed in FIFO (First-In, First-Out) order.

## What is Continuous Integration (CI) and Continuous Delivery (CD)?

* **CI:** The practice of automating the process of building, testing, and integrating code into a central repository.
* **CD:** The practice of automating the deployment of tested and validated code into different production environments.
